# Versioning Policy

Structora Core uses semantic versioning expectations.

Current release: `v0.1.0-alpha`

## Alpha

Alpha releases are public but still evolving. APIs and schema may change, but changes must be documented and tested.

## Beta

Beta releases should reduce schema churn and provide migration notes for public contract changes.

## Stable

Stable releases should preserve backwards compatibility within a major version.

## Schema Versioning

Every discovery result includes `schema_version`. The first public schema is `0.1.0-alpha`.
